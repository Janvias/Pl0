## 1. 系统概述

### 1.1 项目定位

本项目实现了一个完整的 PL/0 语言编译器，采用 **双验证语法分析架构**（LL(1) 递归下降 + LR(1) 规范分析），两个解析器并行运行并交叉验证结果，远超出课程设计的基本要求。

### 1.2 模块组成

```
源程序 → 词法分析 (Lexer) → 双通道语法分析 → 语义分析 & 四元式生成
                              ├─ LL(1) 递归下降
                              └─ LR(1) 移进-归约 (382状态)
                                     ↓
                              交叉验证 (compareParserResults)
```

### 1.3 技术规格

| 项目 | 说明 |
|------|------|
| 语言标准 | C++17 |
| 编译器 | MinGW g++ / GCC |
| 代码规模 | ~5,700行（33个源文件） |
| 文法产生式 | 46条（LR(1)文法） |
| LR(1)状态数 | 382个 |
| 关键字 | 13个（const/var/procedure/begin/end/if/then/while/do/call/read/write/odd） |
| 运算符 | 14个（+/-/*/ / =/#/</>/<=/>=/:=/<>） |
| 操作码 | 18种（ADD/SUB/MUL/DIV/ODD/EQ/NEQ/LT/LTE/GT/GTE/ASSIGN/JUMP/JZ/JNZ/CALL/RET/SYSS/SYSC/READ/WRITE） |

### 3.4 双验证架构（核心创新点）

编译器支持三种运行模式，通过命令行参数切换：

| 模式 | 命令 | 功能 |
|------|------|------|
| LL(1) ONLY | `--ll1` | 仅运行递归下降分析器 |
| LR(1) ONLY | `--lr1` | 仅运行LR(1)移进-归约分析器（382状态） |
| DUAL | `--dual`（默认） | 两个分析器并行运行，比较结果——**解析成功即视为验证通过**，四元式差异仅作信息输出 |

**DUAL模式下LL(1)与LR(1)四元式差异说明**（已知设计差异，非缺陷）：
- LL(1)采用L-翻译模式，LR(1)采用S-翻译模式
- LL(1)使用隐式栈传递中间值，LR(1)使用显式临时变量（T1, T2, ...）
- 两个分析器生成的四元式语义等价，仅编码策略不同

### 4.3 符号表实现

- **数据结构**：`Scope` 栈（`vector<Scope>`），每个 `Scope` 含 `unordered_map<string, Symbol>`
- **作用域**：词法（静态）作用域，支持进入/退出过程体
- **查找**：从最内层向外层搜索（`lookup`）
- **重复检测**：`addSymbol` 在同一作用域内检查重复，返回 `false` 表示重复声明错误
- **临时变量**：`getNewTemp()` 计数器生成 T1→Tn
